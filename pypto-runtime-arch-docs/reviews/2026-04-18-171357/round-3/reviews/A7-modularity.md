## 2. Pros (additions from stress round)

- **Amended A7-P4 (§5) now explicitly anchors A2-P6, A6-P3, A1-P6 inside `distributed/`.** Stress-replay of 6.1.2 step 4 (`REMOTE_SUBMIT` via Horizontal Channel) shows `transport/` only sees `MessageHeader + MessageType + framing`. Parsing, dispatch, bounds checks, descriptor-template caching all happen in `distributed/protocol_handler`. SRP boundary survives the replay — one clean cut (D3, D4, D5).
- **A7-P2 + A9-P1 merger survives the "11th consumer" stress probe.** A hypothetical `SchedulerInspector` needing `Wiring + Completion` but not `Submit` links against `ISchedulerWiring.h` and `ISchedulerCompletion.h` only. The aggregator `ISchedulerLayer` stays unreferenced by that consumer — so the narrow consumer surface is genuinely narrow at the compile-unit level, not just at the conceptual level. ISP win (D4, D2, G2).
- **A8-P1 (`IClock`) + A8-P2 (`IEventLoopDriver` single seam) + A8-P11 (HAL contract tests)** close the X5 ↔ D2 loop at module boundaries. Replay 6.1.1 step 2 under a `StubClock + RecordedEventSource`: the scheduler compiles against `hal/` abstractions only and the unit test does not link `hal/ascend_impl/`. Zero leak — DIP holds (D2).
- **A5-P2 vs A5-P9 module split (peer-health in `distributed/`, worker-health in `scheduler/`) survived the "A6-P6 audit trail + A8-P4 dump_state" combined stress**: each module answers `describe()` and emits `IAuditSink` records *within its own boundary* — no centralized `SystemState` god-object materializes. D5 high-cohesion / D4 narrow-coupling both hold.

## 3. Cons (additions from stress round)

- **Two edges of the handler-registry placement remain only by amendment**: A2-P6 places the registry in `distributed/`, but the round-2 §(A) amended text still reads "registry dispatch" without pinning the owner module. If the eventual edit lands in `transport/` (out of inertia), A7-P4's SRP gain reverses. **Mitigation is in §5 below** — A7-P4 restated with explicit file paths and an invariant "the registry header must not be includable from `transport/`".
- **A7-P6 (sub-namespace `runtime::composition`) still lacks an ADR** despite A4-P6's back-link rule. The stress-replay of 6.1.1 step 1 (`bindings/` → `runtime/`) shows that `runtime::composition` has a distinctly different "reason to change" (deployment-YAML dialect + MLR schema) from the rest of `runtime/` (lifecycle + orchestration). Without an ADR the D1 cohesion argument for keeping it *inside* `runtime/` as a sub-namespace (rather than a separate module) is fragile. Amendment in §5.
- **A2-P6's `IDistributedProtocolHandler` + A2-P3's "open extension-point enums + closed `DepMode`"** together define a *policy axis* across modules. Stress replay against 6.2.4 (RDMA timeout) showed that `DepMode` enum closedness depends on `distributed/` not smuggling in a new DepMode variant via the handler registry. **This is not an A2 concern to resolve — it is an A7 concern**: role-separation between "open-enum extension" (policy plugin) and "closed-enum wire invariant" (hot path) must be policed at the D6 layering boundary. Escalated as a new note in §7 tensions; no new proposal needed.
- **Audit trail sink placement (A6-P6) into `profiling/`** survives the cohesion check but increases `profiling/`'s per-interface stability class count: `ILogSink`, `IMetricsSink`, `IAuditSink`, `ITraceSink`, `IFaultInjector` (sim-only). Stress: at N=5 sinks the module still has one reason to change ("how events leave the runtime"). At N=6+ this would fail D5. Noted; **BC policy (A2-P5) must include a stability class per sink** so the split direction (keep together / fission) is decidable later.
