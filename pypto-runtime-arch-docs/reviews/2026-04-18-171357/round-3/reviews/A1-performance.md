## 2. Pros (carry-over + stress findings)

- Round-2 consensus is structurally sound on the hot path: every HPI-flagged peer proposal replays cleanly through Scenario 6.1.1 Steps 1–11 without violating §4.8.1 (`04-process-view.md:641-653`), §4.8.3 (`04-process-view.md:668-679`), or §4.8.4 (`04-process-view.md:681-694`).
- Scenario 6.2.1 replay (AICore hang, `06-scenario-view.md:83-98`) shows that A5-P6 (watchdog) + A8-P4 (`dump_state`) + A3-P4 (`DEP_FAILED` + successor walk) compose into a coherent failure-path evidence stack **without** introducing an atomic store on the success hot path, when the deadman write is folded into the existing per-cycle timestamp capture (§7 of R2).
- A9-P2's R2 amendment ("single `IEventLoopDriver` test-only seam + closed policy enums") is, on replay, a hot-path **win**: Stage B (§4.8.1 Row "Host Scheduler: submit → DEP_READY") drops two `IEventCollectionPolicy`/`IExecutionPolicy` virtual calls and replaces them with a compiler-visible enum switch — the aggregated `DEP_READY → DISPATCHED` instruction footprint shrinks enough that the 5 μs stage (§4.8.1 Row 3) regains measurable margin.
- A9-P6's option (iii) landing zone (FUNCTIONAL-only impl + keep enum + keep scaffolding for REPLAY) is hot-path-neutral by construction — sim modes are compiled out at L1, and the REPLAY scaffolding adds no runtime cost unless `PROFILING_LEVEL ≥ L2`. A1 has no performance preference between options (i), (ii), (iii); the synthesizer's recommended (iii) is acceptable.

## 3. Cons (residual, stress-surfaced)

Three residual gaps surfaced during stress replay. Each is actionable by a **minor** amendment to an already-agreed proposal (not a veto). Details in §8; summary here:

- **A1-P6 replay gap (Scenario 6.1.2 aliasing via 6.1.1):** the "descriptor-template dedup" fast path requires a `descriptor_template_id` pre-registration protocol that is currently documented as "register-on-first-use" — but the first-use path is **not** on Scenario 6.1.1 (which is single-node). The single-node replay therefore does not stress the template-cache miss path. A1-P6's per-peer template cache miss cost needs an explicit slow-path budget row in §4.8.2 to keep it verifiable; otherwise the proposal's claim "< 1 KB/wire amortized" is a steady-state-only claim. Amendment below in §5.
- **A5-P6 deadman elision not yet normative:** Scenario 6.1.1 replay (steps 2, 5) shows that the event-loop's per-cycle timestamp capture (A1-P5 + A8-P12) can absorb the deadman write at zero cost — but `modules/scheduler.md` and `02-logical-view/02-scheduler.md §2.1.3.5` do not yet specify this elision. Without the normative text, an implementer could add a second atomic store and silently regress the Stage B 1 μs budget (§4.8.1 Row 2). Amendment requested below in §7.
- **A8-P3 histogram insert placement:** Scenario 6.1.1 Step 10 (`on_submission_retired`, `04-process-view.md:299`) fires the histogram insert on the completion path. R2 amendment committed "< 5 ns branchless increment", but did not pin the histogram bucket array to the CONTROL region (A1-P8 / A1-P14 discipline). If the histogram backing is in DATA or default host heap, the first cache-line pull competes with the admission-queue drain (§4.8.1 Row "retirement"). Amendment request to A8 below in §7 to add the region-placement line.

None of the three are blocking; each is a one-line `modules/*.md` addition by the respective owner.
